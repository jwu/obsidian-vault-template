## 提交信息格式

每一个提交包含有 header，body 和 footer。其格式如下：

```
<type> <subject>

<body>

<footer>
```

其中 header 为必须的，body 和 footer 可以根据提交内容需求来添加。

## Type

常见 Type 种类

- 代码类
    - **feature** 一个新功能
    - **fix** 一个 bug 修复
    - **doc** 文档更新
    - **style** 代码规范修改
    - **refactor** 代码重构
    - **perf** 性能优化
    - **test** 测试代码，测试样例等
    - **misc** 工程杂项
    - **submodule** Git 子模块更新
- 3D 资源类
    - **scene** 场景资源提交
    - **prefab** prefab 资源提交
    - **tex** 贴图资源提交
    - **shd** Shader 资源提交
    - **mat** 材质球资源提交
    - **model** 模型资源提交
    - **anim** 动画资源提交
    - **sfx** 特效资源提交
    - **ui** ui 资源提交
    - **sky** 天空盒资源提交
    - **col** 碰撞体资源提交
    - **bake** 场景烘焙资源提交
    - **script** 美术用脚本资源提交
    - **vendor** 第三方/外部资源提交
- UI 资源类
    - **bmf** BitmapFont 字体资源提交
    - **skh** Sketch 资源提交
    - **psd** Psd 资源提交
    - **jpg** Jpg 格式提交
    - **png** Png 格式提交
    - **s9** 切图资源提交
- 策划类
    - **param** 参数提交
- 原画类
    - **chr** 角色资源提交
    - **env** 场景资源提交
    - **item** 物件资源提交

## Subject

标题内容包含了对本次提交的简要描述（注意，标题内容不宜过长，请在 50 个字内表述）:

- 使用 “添加”，“加入”，“修改”，“更新”，“修正”，“调整” 等词汇来表述。
- **不要**在标题末尾使用句号。

## Body

当你的修改描述无法在标题内容内说明清楚，请在 Body 中进行详细说明。

## Footer

下标一般针对重要更新。如果某个更新会导致程序其他人冲突或带来架构或工程上的巨大变动，请在下标处说明。

## 参考样例

```
<feature> 加入技能特效
<fix> 修正了技能释放后没有声音的问题
<docs> 添加主战场逻辑的程序设计文档
```
